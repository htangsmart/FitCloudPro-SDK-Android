AbsTelephonyControl

abstract class AbsTelephonyControl<T : AbsPhoneStateListener>(context: Context, connector: FcConnector, factory: PhoneStateListenerFactory<T>)

这个类内部完成了对电话的处理，注意外部必须有地方持有这个对象的引用，因为PhoneStateListener是在系统内部是弱引用持有的，外部必须保证不被垃圾回收，否则PhoneStateListener就会失效了。 参考：https://stackoverflow.com/questions/40183905/telephonymanagers-phonestatelistener-is-not-called-on-nougat

必须需要权限 监听电话 Manifest.permission.READ_PHONE_STATE 挂断电话 Manifest.permission.CALL_PHONE 可选，查询联系人名称 Manifest.permission.READ_CONTACTS

ToNote:已知的bug:

  1. 因为SubscriptionManager.getActiveSubscriptionInfoList必须要有Manifest.permission.READ_PHONE_STATE才能获取到值。 而部分手机(比如小米)此权限进行了魔改，可能返回空白权限，这个在APP里无法判断，也无法监听。如果手动在系统设置改了权限，需要重启APP才能生效

  2. 因为没有使用SubscriptionManager.addOnSubscriptionsChangedListener监听改变。所以如果双卡双待手机，由一个卡变成两个卡，可能需要重启才能正常监听到两个电话卡的来电
