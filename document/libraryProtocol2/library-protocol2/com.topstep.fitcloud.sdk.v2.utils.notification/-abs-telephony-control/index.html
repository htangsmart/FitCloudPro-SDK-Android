AbsTelephonyControl

abstract class AbsTelephonyControl<T : AbsPhoneStateListener>(context: Context, connector: FcConnector, factory: PhoneStateListenerFactory<T>)

Monitoring and handling of telephony.

External must hold a reference to this object, Because PhoneStateListener is held by Weak reference inside the system, the external must be guaranteed not to be garbage collected, otherwise PhoneStateListener will become invalid

See alsoï¼šhttps://stackoverflow.com/questions/40183905/telephonymanagers-phonestatelistener-is-not-called-on-nougat

Permissions:

  • monitor: Manifest.permission.READ_PHONE_STATE

  • hang up: Manifest.permission.CALL_PHONE

  • query contact name: Manifest.permission.READ_CONTACTS

  • send sms: Manifest.permission.SEND_SMS

ToNote:Known bugs:

  1. Because SubscriptionManager.getActiveSubscriptionInfoList must have Manifest.permission.READ_PHONE_STATE to obtain a value. Some mobile phones (such as Xiaomi) may return blank permissions, which cannot be determined or monitored in the app. If you manually change the permissions in the system settings, you need to restart the app to take effect

  2. Because SubscriptionManager.addOnSubscriptionsChangedListener was not used to listen for changes. So if a dual card dual standby phone changes from one card to two cards, it may need to be restarted to properly listen for incoming calls from both phone cards
