FcSDK

interface FcSDK

The FitCloudSDK Entry Class.

You need to perform the following configuration for init:

1.Configure log

The FitCloudSDK use Timber to output log, so you need to configure it. And the BLE function is based on RxAndroidBLE, you can use RxBleClient.updateLogOptions to customize log output.

2.Use FcSDK.Builder to create a FcSDK instance

Singleton pattern is recommended

3.Handle App UI Process state and use FcSDK.isForeground to set

Such as:

ProcessLifecycleOwner.get().lifecycle.addObserver(object : DefaultLifecycleObserver {
  override fun onStart(owner: LifecycleOwner) {
      fcSDK.isForeground = true
  }

  override fun onStop(owner: LifecycleOwner) {
      fcSDK.isForeground = false
  }
})

4.Set RxJavaPlugins.setErrorHandler to ignore some undeliverable exception

It is recommended to ignore exceptions in FcSDK.rxJavaPluginsIgnoreExceptions

5.Make sure FcSDK.connector create in main thread


After init, you can use FcSDK.scanner to scan device. And use FcSDK.connector to connect and control device.
