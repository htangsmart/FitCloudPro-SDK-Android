val storageAddr: Int

条目文件在flash中的存储位置，在OTA工具打包时可指定文件存储在哪个地址段中， 若为0，则表示按固件中原有的 mft 定义的分区表地址；不为0时则以该地址段为主； 这样设计是由于当固件、mft、 UI、Font等文件统一打包后升级，原有的mft分区有 可能已被修改了，但新的mft尚未更新，此时需要按照该地址中指定的地址存储； 以前旧的OTA只能每个文件按顺序升级，先升mft，再升UI，所以无法打包成整体
